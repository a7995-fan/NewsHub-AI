# AI问答功能使用说明

## 功能概述

本项目集成了千问大模型，实现了基于数据库信息的智能问答功能。AI助手可以：
- 基于数据库中的新闻信息回答用户问题
- 搜索相关新闻并提供基于搜索结果的回答
- 保持对话上下文，支持连续对话
- 引用数据库中的新闻标题和内容

## 配置步骤

### 1. 获取千问API密钥

1. 访问阿里云千问平台：https://dashscope.aliyuncs.com/
2. 注册/登录账号
3. 在控制台创建API密钥
4. 复制API密钥

### 2. 配置环境变量

1. 复制 `.env.example` 文件为 `.env`：
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，填入您的千问API密钥：
   ```env
   QIANWEN_API_KEY=your_qianwen_api_key_here
   QIANWEN_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
   ```

### 3. 安装依赖

依赖包已经在 `requirements.txt` 中添加，运行以下命令安装：
```bash
pip install -r requirements.txt
```

## 功能说明

### API接口

#### 1. AI问答接口
- **路径**: `POST /api/ai/chat`
- **参数**:
  - `message`: 用户消息（必填）
  - `conversation_id`: 会话ID（可选，用于保持上下文）
- **返回**: AI回复消息和会话ID

#### 2. 基于搜索的AI问答接口
- **路径**: `POST /api/ai/chat/search`
- **参数**:
  - `message`: 用户消息（必填）
  - `conversation_id`: 会话ID（可选）
- **返回**: 基于搜索结果的AI回复

#### 3. 获取AI分类接口
- **路径**: `GET /api/ai/categories`
- **返回**: 新闻分类列表

#### 4. 获取AI新闻接口
- **路径**: `GET /api/ai/news`
- **参数**:
  - `limit`: 返回新闻数量（默认20）
- **返回**: 新闻列表

### 前端使用

前端页面位于 `frontend/src/views/AI.vue`，功能包括：
- 聊天界面
- 消息历史显示
- 自动滚动到底部
- 加载状态显示
- 消息格式化（支持换行、粗体、斜体）

## 技术实现

### 后端技术栈
- **LangChain**: 用于构建AI对话链
- **千问模型**: 阿里云的大语言模型
- **FastAPI**: 后端API框架
- **SQLAlchemy**: 数据库ORM

### 前端技术栈
- **Vue3**: 前端框架
- **Element Plus**: UI组件库
- **Axios**: HTTP请求库

### AI功能实现
1. **数据库信息获取**: 从数据库中获取新闻分类和新闻信息
2. **信息格式化**: 将数据库信息格式化为AI可理解的文本
3. **提示工程**: 设计专业的新闻助手提示模板
4. **对话管理**: 使用ConversationBufferMemory保持对话上下文
5. **搜索增强**: 基于关键词搜索相关新闻，提供更精准的回答

## 注意事项

1. **API密钥安全**: 请勿将 `.env` 文件提交到版本控制系统
2. **API调用限制**: 千问API有调用次数限制，请注意控制使用频率
3. **回答准确性**: AI回答基于数据库信息，如果数据库中没有相关信息，AI会如实告知
4. **性能优化**: AI问答需要调用外部API，响应时间可能较长，建议在前端添加加载状态

## 故障排除

### 问题1：AI回复失败
**可能原因**:
- API密钥配置错误
- 网络连接问题
- API调用次数超限

**解决方法**:
- 检查 `.env` 文件中的API密钥是否正确
- 检查网络连接
- 查看阿里云控制台的API使用情况

### 问题2：回答不准确
**可能原因**:
- 数据库中相关信息不足
- 提示模板需要优化

**解决方法**:
- 运行数据填充脚本，添加更多新闻数据
- 优化提示模板，提高回答质量

### 问题3：前端无法连接后端
**可能原因**:
- 后端服务未启动
- 跨域配置问题

**解决方法**:
- 启动后端服务：`uvicorn main:app --reload`
- 检查CORS配置
